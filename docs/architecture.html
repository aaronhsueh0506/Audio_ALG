1. 整體架構與訊號流

Pipeline 共四個處理階段，每 frame 處理 10 ms（160 samples @ 16 kHz）：

圖 1：整體 pipeline 概觀
關鍵點：
  1. HPF 同時對 mic 和 ref 做（保持相位一致）
  2. AEC 只做線性消除，輸出 error + context（不含 RES）
  3. NR 處理 error 訊號，輸出去噪結果與 per-bin gain
  4. RES 用 NR 的 gain 修正 echo PSD，做最後一層 residual 抑制

2. HPF 高通濾波器

目的：移除 DC offset、低頻 rumble、50/60 Hz 電源 hum，避免污染 AEC 自適應演算法。

圖 2：HPF 頻率響應 — 80Hz 以下大幅衰減，語音頻段透通
項目規格
類型2nd-order Butterworth (IIR)
截止頻率80 Hz (−3 dB)
實作Direct Form II Transposed
位置應用層 (Audio_ALG/lib/hpf)，AEC module 不含
注意：v2.0.0 之後 HPF 從 AEC module 移到應用層，由 pipeline 對 mic / ref 各自呼叫一次。

3. AEC — PBFDKF + Shadow Filter

核心是 PBFDKF (Partitioned Block Frequency-Domain Kalman Filter)，搭配 shadow filter 做雙濾波器策略。

3.1 PBFDKF 架構

圖 3：PBFDKF 結構 — 頻域 partition 卷積 + Kalman 權重更新
為什麼用 PBFDKF？
  • 頻域處理：FFT 加速卷積，降低複雜度
  • Partition：把長 filter 切成多個短 partition，減少 latency
  • Kalman：動態調整 step size（Q/R），比固定 mu 的 NLMS 收斂更快、追蹤更穩
項目值
block_size320 samples (= 2 × hop)
fft_size512 (next pow2)
n_partitionsfilter_length ÷ hop ≈ 4 (預設 FL=512)
Kalman Q (process noise)1e−3 (high), 1e−5 (low)
P_MAX0.5 (state covariance ceiling)

3.2 Shadow Filter 雙濾波器策略

圖 4：Shadow Filter 雙濾波器策略 — main 穩定輸出, shadow 快速追蹤
為什麼要兩個 filter？

單一 filter 必須在「快速追蹤 echo path 變化」和「DT 段抗污染」之間取捨。雙 filter 各司其職，再用 bidirectional copy 自動把較好的權重複製過去，兩全其美。

3.3 Echo Path Change (EPC) 偵測

當 main + shadow 兩者誤差同時上升、且差距變小時，視為 echo path 變化（移動、新的反射路徑等）：

errors_rising = (total_err > prev_total_err × 1.5)  &  delta_ratio < 0.3

偵測到 EPC 後：

  • Q 暫時拉高 (Q_high) 加速 re-convergence
  • P_MAX 暫時放寬
  • filter_converged 設為 False
  • 20 frame hangover 期間 RES 進入保守模式

3.4 Stationary far-end DT 保護 (v13)

問題場景：白噪音、風扇、冷氣等穩態 far-end + 近端有人講話 → 傳統 DT 偵測全部失效（dt_indicator → 0, coh² → 1），RES 把語音當 echo 壓掉。

圖 5：Stationary DT 兩階段偵測 + 四重保護
效果：合成 WN DT 測試，語音保留率從 5% → 48%（接近 NoRES baseline 50%）。
800-case blind test 一般語音場景零退步（所有改動被 is_stationary_dt 守門）。

4. NR — SPP + OMLSA

NR 處理 AEC 輸出的 error 訊號，移除背景噪音同時保留近端語音。三大組件：MCRA + SPP + MMSE-LSA。

圖 6：NR 三組件協作流程

4.1 MCRA — 噪音追蹤

用 minimum tracking 估計噪音 PSD，假設噪音在約 320 ms 視窗內統計穩定：

P[k,n] = αp · P[k,n−1] + (1−αp) · |Y[k,n]|²
Pmin[k] = min over L=32 frames
Sr = P / Pmin < δ → 更新 noise_psd

4.2 SPP — 語音存在機率

對每個 frequency bin 估計「該 bin 是語音」的後驗機率：

γ[k] = |Y[k]|² / noise_psd[k]   (a posteriori SNR)
ξ[k] = α · ξprev + (1−α) · max(γ−1, 0)   (decision-directed)
p[k] = 1 / (1 + (1+ξ) · e−v · q/(1−q))

4.3 MMSE-LSA — 對數頻譜 MMSE 增益

Ephraim & Malah 1985 的對數頻域 MMSE 估計，加上 SPP 加權（OMLSA）：

GLSA[k] = (ξ / (1+ξ)) · exp(½ ∫v∞ e−t/t dt)
G[k] = GLSAp[k] · gmin1−p[k]

5. Post-filter — RES + CNG

RES 處理 NR 後的訊號，把 linear AEC 漏的 residual echo 進一步壓掉。CNG 在純 FS 段補上舒適噪音避免突然安靜。

5.1 ENR (Echo-to-Nearend Ratio) 計算

圖 7：RES 內 ENR 計算流程 — 用 NR 的 G_NR² 修正 echo PSD，再除以 nearend_est 得 ENR
GRES[k] = clamp((enrs − ENR) / (enrs − enrt),   gmin,   1.0)

FS 段（dt 低）用較嚴格的 enr_t，DT 段（dt 高）用較寬鬆的 enr_t（two-tuning）。

6.5 NR 降噪等級

NR 透過 g_min_db 控制最大抑制量（語音/噪音比值的下限），搭配 alpha_g 控制時域平滑：

等級g_min_db最大抑制量alpha_g推薦場景
Light −15 dB 15 dB 0.70 高 SNR (>10 dB)、近端品質優先、保留細節
Balanced −20 dB 20 dB 0.85 預設、一般場景
Aggressive −25 dB 25 dB 0.90 低 SNR (<5 dB)、需要安靜背景
等級選擇 trade-off：
  • Light：語音失真最小，但靜音段仍聽得到底噪
  • Balanced：echo/quality 平衡，適合通用語音場景
  • Aggressive：背景幾乎靜音，但低 SNR 下語音可能有 musical noise / 鈍化

詳細參數調整指南見 NR/parameter_adjust_guide.md。

7. 已知限制

類別限制
輸入 單通道、8 / 16 / 48 kHz，無 spatial filtering
AEC 線性模型 非線性 echo（speaker 飽和）靠 RES 壓制，無獨立非線性模型
AEC 穩態 far-end 弱信號段 linear AEC 仍可能在弱信號段把語音吸收（合成測試 ~10% 段）
AEC delay estimation GCC-PHAT 在低 SNR 或強非線性場景可能不準
NR 突發噪音 impulse / crackle / 敲擊 — MCRA TC 跟不上
NR 噪音突升 / 突降 0.3-0.6 s 內噪音漏抑制（minimum tracking 需收斂）
NR 音樂背景 SPP 二元假設（語音 vs 噪音），音樂被誤判為語音保留
NR 強迴響 reverb tail 與穩態噪音相近，可能被誤抑制
RES musical noise 低 SNR 下仍可能殘留
